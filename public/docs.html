WhatsApp Gateway ArtaPay API

Panduan integrasi cepat untuk menghubungkan kirim pesan WhatsApp ke sistem lain dengan API key per device.

All endpoints (except GET /api/status and GET /api/qr) require the header X-API-Key: <your-key>
Cara Integrasi
1. Pastikan device sudah ready.
2. Ambil API key per device dari menu device/dashboard.
3. Gunakan endpoint POST /api/send untuk teks.
4. Gunakan POST /api/send/media untuk media.
5. Tambahkan device_id bila integrasi Anda menangani banyak device.
Contoh Header
Base URL  : http://localhost:3000
X-API-Key : YOUR_API_KEY

POST /api/send
Content-Type: application/json
Contoh Integrasi
curl -X POST "http://localhost:3000/api/send" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: YOUR_API_KEY" \
  -d '{"to":"628123456789","message":"Halo dari ArtaPay"}'
fetch('http://localhost:3000/api/send', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'X-API-Key': 'YOUR_API_KEY'
  },
  body: JSON.stringify({ to: '628123456789', message: 'Halo dari JS' })
});
$payload = ['to' => '628123456789', 'message' => 'Halo dari PHP'];
requests.post(
  'http://localhost:3000/api/send',
  headers={'X-API-Key': 'YOUR_API_KEY'},
  json={'to': '628123456789', 'message': 'Halo dari Python'}
)
req.Header.Set("X-API-Key", "YOUR_API_KEY")
req.Header.Set("Content-Type", "application/json")
POST /api/send/media
{
  "to": "628123456789",
  "mediaUrl": "https://example.com/file.pdf",
  "caption": "Dokumen ArtaPay"
}

Status & QR

GET /api/status Get device connection status
Auth Required
No
Method
GET
Response
{ "success": true, "data": { "status": "ready", "info": { "pushname": "...", "number": "..." } } }
GET /api/qr Get QR code (base64) for scanning

Returns the current QR code as a base64 data URI. Auth not required.

Contacts & Chats

GET /api/chats Get recent chats
GET /api/contacts Get all contacts
GET /api/messages/:chatId Get messages from a chat
GET /api/check/:number Check if number is on WhatsApp

Webhooks — per device_id support. Use * to receive all devices.

GET /api/webhooks List all webhooks
POST /api/webhooks Register a new webhook
Payload
POST /api/webhooks

{
  "url": "https://…/hook",
  "device_id": "*",
  "events": ["message","message_sent"],
  "description": "My hook"
}

Available events:
• message         • message_sent
• message_ack     • typing
• connected       • disconnected
• qr              • *  (all)
DELETE /api/webhooks/:id Delete a webhook
PATCH /api/webhooks/:id/toggle Enable / disable webhook

Device Control

POST /api/logout Logout from WhatsApp

⚠ This will disconnect the WhatsApp session.

POST /api/restart Restart WhatsApp service

Webhook Payload Reference

Every webhook delivery sends a POST with this structure:

{
  "event":     "message",
  "device_id": "wa-gateway-session",
  "timestamp": "2025-01-01T00:00:00.000Z",
  "data": {
    "id":       "XXXXXXX",
    "from":     "628123456789@c.us",
    "to":       "628987654321@c.us",
    "body":     "Hello!",
    "type":     "chat",
    "isGroup":  false,
    "timestamp": 1700000000
  }
}
Security — HMAC Signature

If a secret is set on the webhook, the delivery includes:

X-WA-Signature: sha256=HMAC(secret, body)
