Integrasi

Panduan implementasi kirim teks WhatsApp ke sistem lain dengan contoh kode siap pakai dan tester realtime.

API Key Integrasi — not set
Header wajib: X-API-Key: <key>

cURL - Contoh Implementasi

Header dan payload disesuaikan dengan endpoint `POST /api/send`.

Format Request

Pola dasar yang dipakai semua contoh implementasi.

Header wajib
Content-Type: application/json
X-API-Key: YOUR_API_KEY
Payload wajib
to = nomor tujuan, contoh 628123456789
message = isi pesan teks yang akan dikirim
Payload opsional
device_id = jika ingin pakai device tertentu
quotedMessageId = jika ingin reply ke pesan tertentu
Preview Payload
{
  "to": "628123456789",
  "message": "Halo dari integrasi sistem saya"
}

Terima Pesan via Webhook

Cara paling realtime untuk menerima pesan masuk dari WhatsApp adalah mendaftarkan webhook.

Langkah singkat
1. Siapkan endpoint publik, misalnya https://domainanda.com/webhook/wa
2. Register webhook ke POST /api/webhooks
3. Pilih event message atau message,message_sent
4. Saat ada pesan masuk, server Anda akan menerima payload JSON realtime
Request register webhook
POST /api/webhooks
X-API-Key: YOUR_API_KEY
Content-Type: application/json

{
  "url": "https://domainanda.com/webhook/wa",
  "device_id": "device-001",
  "events": ["message"],
  "description": "Webhook pesan masuk"
}
Contoh payload pesan masuk
{
  "event": "message",
  "device_id": "device-001",
  "timestamp": "2026-03-29T15:56:00.000Z",
  "data": {
    "id": "3EB0ABC123",
    "from": "628123456789@c.us",
    "from_num": "628123456789",
    "name": "Customer A",
    "body": "Halo, saya mau tanya produk",
    "type": "chat",
    "isGroup": false
  }
}
Terima Pesan via Polling

Kalau belum siap pakai webhook, Anda bisa membaca chat dan message secara berkala lewat API.

Endpoint yang dipakai
GET /api/chats untuk daftar chat terbaru
GET /api/messages/:chatId untuk baca isi chat
GET /api/contacts untuk sinkron data kontak
Contoh ambil daftar chat
GET /api/chats?limit=20
X-API-Key: YOUR_API_KEY
Contoh ambil pesan per chat
GET /api/messages/628123456789%40c.us?limit=20
X-API-Key: YOUR_API_KEY
Contoh response message list
{
  "success": true,
  "data": [
    {
      "id": "3EB0ABC123",
      "from": "628123456789@c.us",
      "body": "Halo admin",
      "type": "chat",
      "timestamp": 1711700000,
      "fromMe": false
    }
  ]
}
Untuk sistem yang butuh update realtime, **webhook lebih disarankan** daripada polling.
